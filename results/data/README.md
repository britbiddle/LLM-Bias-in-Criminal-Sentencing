# `results/data/` — paper data

Everything here is regenerated by `notebooks/ablation_study.ipynb`. Filenames match the in-notebook variable names where possible.

## Source of truth

### `study_v2_results.parquet` (1.7 MB, 13,200 rows)
**Every LLM prediction from the main study.** Load this to reproduce any analysis. One row per `(case_id, model, condition)` triple.

Schema:
| Column | Type | What |
|---|---|---|
| `case_id` | int | JUSTFAIR row index; joins back to `data/justfair_drugs_filtered.parquet` |
| `model` | str | `claude-haiku-4-5`, `gpt-4o-mini`, `qwen-2.5-7b`, or `llama-3.2-3b` |
| `condition` | str | one of 11: `baseline`, `race_{W,B,H}`, `race_debias_{W,B,H}`, `name_{W,B,H}`, `ses` |
| `predicted_months` | int | the LLM's answer, post-JSON-parse |
| `reasoning` | str | the LLM's rationale (2–4 sentences, from the JSON `reasoning` field) |
| `parse_error` | str or null | null if the JSON parsed cleanly (100% of this run) |
| `real_race`, `real_sentence` | str, int | what actually happened for this defendant |
| `CHAP2`, `CRIMPTS`, `STATMIN`, `STATMAX`, `NOCOUNTS` | num | USSC case covariates |
| `MONSEX`, `PRESENT`, `NEWCNVTN`, `WEAPON`, `CAROFFAP` | various | other covariates used by the prompt |
| `pred_clip` | int | `predicted_months` clipped to `[0, STATMAX]`; this is what analyses use |

### `pilot_results.parquet` (1.0 MB)
Older pilot from an earlier notebook revision. **Don't use for the paper** — kept for provenance only. Schema partially overlaps with `study_v2_results.parquet` but with fewer conditions.

### `prompt_sensitivity.parquet` (53 KB, 150 rows)
Output of the prompt-sensitivity appendix. Three alternate templates × 50 cases × gpt-4o-mini only. Schema is the same as `study_v2_results.parquet` with an additional `template` column (`primary (decoded)`, `concise`, `verbose`).

## Derived analysis tables (all tiny, all committed)

### `race_coefficients.csv` / `.jsonl` (2.2 KB / 4.9 KB, 17 rows)
**The rigor chart data.** Black-vs-White and Hispanic-vs-White OLS coefficients from every condition family, plus the real-judge benchmark.

Columns: `source`, `condition_type`, `race_vs_white`, `coef_months`, `ci_lo`, `ci_hi`, `p`.

Generated in notebook cell 42. This is the canonical table for `fig_unified_forest.png`.

### `robust_stats.csv` (2.2 KB, 24 rows)
**Within-case ablation statistics.** One row per `(ablation, model, comparison)` for `ablation ∈ {race, name}` × 4 models × `comparison ∈ {BW, HW, BH}`.

Columns: `ablation`, `model`, `comparison`, `n`, `mean_delta`, `ci_lo`, `ci_hi`, `cohens_d`, `p_paired_t`, `p_wilcoxon`, `p_tost_pm5`, `equiv_pm5`, `p_paired_t_bonf`, `p_wilcoxon_bonf`, `sig_at_bonf_05`.

Generated in cell 34. This is the "surface bias is null" evidence.

### `debias_shift.csv` (611 B, 12 rows)
Within-case mean shift `(debias_{race} − race_{race})` per model × race.

Columns: `model`, `race`, `n`, `mean_shift`, `ci_lo`, `ci_hi`, `p`.

Generated in cell 60.

### `prompt_sensitivity.csv` / `.jsonl` (121 B / 225 B, 3 rows)
Summary of the Black–White gap across the three prompt templates.

Columns: `template`, `n`, `mean_BW`, `ci_lo`, `ci_hi`.

Generated in cell 65.

## Not committed (regenerable on demand)
